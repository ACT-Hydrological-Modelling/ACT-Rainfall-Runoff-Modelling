## 1. Introduction

### 1.1 Purpose

This document specifies the design and implementation requirements for integrating a nonlinear Muskingum river routing module into the existing rainfall-runoff modeling library. The module provides optional channel routing capabilities that can be applied to hydrographs generated by rainfall-runoff models, with full support for parameter calibration.

### 1.2 Scope

The specification covers:

- Mathematical formulation of the nonlinear Muskingum method
- Software architecture and class design
- Integration patterns with existing rainfall-runoff models
- Calibration framework integration
- Configuration management
- API design and usage patterns

### 1.3 Design Principles

1. **Optional Integration**: Routing is an opt-in feature that does not affect existing model behavior when disabled
2. **Modularity**: The routing module operates independently and can be used standalone or integrated
3. **Calibration Compatibility**: Routing parameters integrate seamlessly with the existing calibration framework
4. **Minimal Dependencies**: No additional external dependencies beyond NumPy/SciPy
5. **Consistent API**: Follows existing library conventions for model interfaces

### 1.4 Terminology

| Term | Definition |
|------|------------|
| **Reach** | A segment of river/channel through which flow is routed |
| **Sub-reach** | A subdivision of the main reach for numerical purposes |
| **Inflow Hydrograph** | Time series of discharge entering the reach (output from RR model) |
| **Outflow Hydrograph** | Time series of discharge leaving the reach (routed result) |
| **Storage Constant (K)** | Parameter relating storage to outflow; units of time |
| **Nonlinear Exponent (m)** | Dimensionless parameter controlling storage-discharge nonlinearity |

## 2. Mathematical Formulation

### 2.1 Governing Equations

The nonlinear Muskingum method is based on two fundamental equations:

#### 2.1.1 Continuity Equation

$$\frac{dS}{dt} = I(t) - Q(t)$$

Where:
- $S$ = storage volume in the reach [L³]
- $I(t)$ = inflow rate [L³/T]
- $Q(t)$ = outflow rate [L³/T]
- $t$ = time [T]

#### 2.1.2 Storage-Discharge Relationship

$$S = K \cdot Q^m$$

Where:
- $K$ = storage constant [T]
- $m$ = nonlinear exponent [dimensionless]
- When $m = 1$: linear reservoir behavior
- When $m < 1$: increased attenuation at lower flows

### 2.2 Combined Differential Equation

Substituting the storage relationship into the continuity equation:

$$\frac{d(K \cdot Q^m)}{dt} = I - Q$$

$$K \cdot m \cdot Q^{m-1} \cdot \frac{dQ}{dt} = I - Q$$

$$\frac{dQ}{dt} = \frac{I - Q}{K \cdot m \cdot Q^{m-1}}$$

### 2.3 Numerical Discretization

Using the trapezoidal rule (Crank-Nicolson scheme) for second-order accuracy:

#### 2.3.1 Discretized Continuity

$$S^{n+1} - S^n = \Delta t \cdot \left( \bar{I} - \bar{Q} \right)$$

Where:
- $\bar{I} = \frac{I^n + I^{n+1}}{2}$ (average inflow over time step)
- $\bar{Q} = \frac{Q^n + Q^{n+1}}{2}$ (average outflow over time step)
- Superscript $n$ denotes time step index

#### 2.3.2 Nonlinear Equation for $Q^{n+1}$

Substituting $S = K \cdot Q^m$:

$$K(Q^{n+1})^m - K(Q^n)^m = \Delta t \left[ \frac{I^n + I^{n+1}}{2} - \frac{Q^n + Q^{n+1}}{2} \right]$$

Rearranging into the form $f(Q^{n+1}) = 0$:

$$f(Q^{n+1}) = K(Q^{n+1})^m + \frac{\Delta t}{2}Q^{n+1} - K(Q^n)^m + \frac{\Delta t}{2}Q^n - \Delta t \cdot \bar{I}$$

Where $\bar{I} = \frac{I^n + I^{n+1}}{2}$

### 2.4 Newton-Raphson Solution

The nonlinear equation is solved iteratively using Newton-Raphson:

$$Q_{k+1} = Q_k - \frac{f(Q_k)}{f'(Q_k)}$$

Where the derivative is:

$$f'(Q) = K \cdot m \cdot Q^{m-1} + \frac{\Delta t}{2}$$

#### 2.4.1 Convergence Criteria

- Absolute tolerance: $|f(Q)| < \epsilon_{abs}$ where $\epsilon_{abs} = 10^{-9}$
- Relative tolerance: $|\Delta Q / Q| < \epsilon_{rel}$ where $\epsilon_{rel} = 10^{-6}$
- Maximum iterations: 50 (with warning if reached)

### 2.5 Reach Subdivision

For improved numerical accuracy, the reach may be divided into $N$ sub-reaches:

#### 2.5.1 Parameter Distribution

$$K_{sub} = \frac{K_{total}}{N}$$

The exponent $m$ remains constant across all sub-reaches (it represents a physical property of the storage-discharge relationship).

#### 2.5.2 Sequential Routing

For each time step, route sequentially through sub-reaches:

```
for each time step n:
    Q_in = I[n]  # External inflow
    for each sub-reach j = 1 to N:
        Q_out[j] = solve_routing(Q_in, K_sub, m, dt)
        Q_in = Q_out[j]  # Cascade to next sub-reach
    final_outflow[n] = Q_out[N]
```

### 3.3 Key Design Patterns

#### 3.3.1 Strategy Pattern for Routing

The routing module uses the Strategy pattern to allow different routing methods:

```python
class BaseRouter(ABC):
    """Abstract base class for all routing methods."""
    
    @abstractmethod
    def route(self, inflow: np.ndarray, dt: float) -> np.ndarray:
        """Route an inflow hydrograph."""
        pass
    
    @abstractmethod
    def get_parameters(self) -> Dict[str, Parameter]:
        """Return calibratable parameters."""
        pass
```

#### 3.3.2 Decorator Pattern for Optional Routing

Routing can be applied as a decorator/wrapper around existing models:

```python
# Without routing
hydrograph = rr_model.simulate(precipitation)

# With routing (wrapper approach)
routed_hydrograph = router.route(rr_model.simulate(precipitation), dt)

# Or using coupled model
coupled = CoupledModel(rr_model, router)
routed_hydrograph = coupled.simulate(precipitation)
```

### 4.2 NonlinearMuskingumRouter Class

Main routing implementation class.

```python
class NonlinearMuskingumRouter(BaseRouter):
    """
    Nonlinear Muskingum river routing with S = K * Q^m relationship.
    
    This router transforms an inflow hydrograph into a routed outflow
    hydrograph by solving the nonlinear storage-discharge relationship
    using numerical methods.
    
    Parameters
    ----------
    K : float
        Storage constant for the entire reach. Units must be consistent
        with the time step used in routing (e.g., hours).
        Typical range: 0.5 - 100 hours depending on reach length.
        
    m : float
        Nonlinear exponent controlling the storage-discharge relationship.
        - m = 1.0: Linear reservoir (storage proportional to discharge)
        - m < 1.0: More attenuation at low flows
        - m > 1.0: More attenuation at high flows (less common)
        Typical range: 0.4 - 1.2, most commonly 0.6 - 1.0
        
    n_segments : int, optional
        Number of sub-reaches for numerical routing. More segments
        provide better numerical accuracy and smoother attenuation.
        Default is 1 (no subdivision).
        Typical range: 1 - 10
        
    solver_config : dict, optional
        Configuration for the Newton-Raphson solver:
        - max_iterations : int (default 50)
        - abs_tolerance : float (default 1e-9)
        - rel_tolerance : float (default 1e-6)
        - min_discharge : float (default 1e-10, prevents division by zero)
    
    Attributes
    ----------
    K_segment : float
        Storage constant per sub-reach (K / n_segments)
        
    Examples
    --------
    >>> router = NonlinearMuskingumRouter(K=5.0, m=0.8, n_segments=3)
    >>> outflow = router.route(inflow_hydrograph, dt=1.0)
    
    Notes
    -----
    The storage constant K can be interpreted approximately as the
    travel time through the reach under steady-state conditions.
    
    For best results, ensure that the time step dt is smaller than K
    to maintain numerical stability.
    """
    
    # Class constants
    DEFAULT_SOLVER_CONFIG = {
        'max_iterations': 50,
        'abs_tolerance': 1e-9,
        'rel_tolerance': 1e-6,
        'min_discharge': 1e-10
    }
    
    def __init__(
        self,
        K: float,
        m: float,
        n_segments: int = 1,
        solver_config: Optional[Dict] = None
    ):
        """Initialize the router with given parameters."""
        pass
    
    # ─────────────────────────────────────────────────────────────
    # Core Methods
    # ─────────────────────────────────────────────────────────────
    
    def route(
        self,
        inflow: np.ndarray,
        dt: float,
        initial_outflow: Optional[float] = None,
        return_intermediate: bool = False
    ) -> Union[np.ndarray, Tuple[np.ndarray, Dict]]:
        """
        Route an inflow hydrograph through the reach.
        
        Parameters
        ----------
        inflow : np.ndarray
            Inflow time series. Shape: (n_timesteps,)
            Units: volumetric flow rate (e.g., m³/s, cfs)
            
        dt : float
            Time step duration. Must be in same units as K.
            
        initial_outflow : float, optional
            Initial outflow condition at t=0. If None, assumes
            steady state (outflow equals first inflow value).
            
        return_intermediate : bool, optional
            If True, also returns dictionary with intermediate results
            including outflow from each sub-reach.
            
        Returns
        -------
        outflow : np.ndarray
            Routed outflow time series. Shape: (n_timesteps,)
            
        intermediate : dict (only if return_intermediate=True)
            Dictionary containing:
            - 'segment_outflows': np.ndarray of shape (n_segments+1, n_timesteps)
            - 'iterations_per_step': list of iteration counts
            - 'convergence_flags': list of convergence status per step
            
        Raises
        ------
        ValueError
            If inflow contains negative values or if parameters are invalid.
        ConvergenceError
            If Newton-Raphson fails to converge for any time step.
            
        Examples
        --------
        >>> inflow = np.array([0, 10, 25, 40, 35, 25, 15, 8, 4, 2])
        >>> router = NonlinearMuskingumRouter(K=2.0, m=0.8)
        >>> outflow = router.route(inflow, dt=1.0)
        """
        pass
    
    def route_steady_state(self, inflow: float) -> float:
        """
        Calculate steady-state outflow for a given constant inflow.
        
        Under steady-state conditions (dS/dt = 0), outflow equals inflow.
        This method is useful for establishing initial conditions.
        
        Parameters
        ----------
        inflow : float
            Constant inflow rate
            
        Returns
        -------
        outflow : float
            Steady-state outflow (equals inflow)
        """

    def calculate_storage(self, outflow: float) -> float:
        """
        Calculate storage volume for a given outflow.
        
        Parameters
        ----------
        outflow : float
            Discharge rate
            
        Returns
        -------
        storage : float
            Storage volume (S = K * Q^m)
        """

    def get_parameters(self) -> Dict[str, Parameter]:
        """
        Return dictionary of calibratable parameters.
        
        Returns
        -------
        parameters : dict
            Dictionary mapping parameter names to Parameter objects:
            - 'routing_K': Storage constant

            - 'routing_n_segments': Number of sub-reaches

        Set parameters from array (for optimizer compatibility).
        
        Parameters
        ----------
        values : np.ndarray
            Array of parameter values
        param_names : list of str
            Parameter names corresponding to values

    # ─────────────────────────────────────────────────────────────
    # Internal Methods
    # ─────────────────────────────────────────────────────────────
    
    def _solve_timestep(
        self,
        I_current: float,
        I_next: float,
        Q_current: float,
        dt: float
    ) -> Tuple[float, int, bool]:
        """
        Solve for outflow at next time step using Newton-Raphson.
        
        Parameters
        ----------
        I_current : float
            Inflow at current time step (n)
        I_next : float
            Inflow at next time step (n+1)
        Q_current : float
            Outflow at current time step (n)
        dt : float
            Time step size
            
        Returns
        -------
        Q_next : float
            Outflow at next time step (n+1)
        iterations : int
            Number of iterations required
        converged : bool
            Whether solution converged within tolerance
        """
        pass
    
    def _routing_function(
        self,
        Q_new: float,
        Q_old: float,
        I_avg: float,
        dt: float
    ) -> float:
        """
        Evaluate the routing equation f(Q_new) = 0.
        
        f(Q) = K*Q^m + (dt/2)*Q - K*Q_old^m + (dt/2)*Q_old - dt*I_avg
        """
        pass
    
    def _routing_derivative(self, Q: float, dt: float) -> float:
        """
        Evaluate derivative of routing function for Newton-Raphson.
        
        f'(Q) = K*m*Q^(m-1) + dt/2
        """
        pass
    
    def _validate_inputs(self, inflow: np.ndarray, dt: float) -> None:

        pass

        return (

    def summary(self) -> str:
        """Return detailed summary of router configuration."""
        pass
    
    def copy(self) -> 'NonlinearMuskingumRouter':
```

## 5. Integration with Rainfall-Runoff Models

### 5.1 Integration Patterns

The module supports multiple integration patterns to accommodate different use cases:

#### 5.1.1 Pattern A: Standalone Routing (Post-Processing)

```python
# Run RR model separately
rr_model = SCSUnitHydrograph(area=100, tc=2.5, dt=1.0)
direct_runoff = rr_model.simulate(precipitation)

# Apply routing as post-processing
router = NonlinearMuskingumRouter(K=5.0, m=0.8, n_segments=2)
routed_outflow = router.route(direct_runoff, dt=1.0)
```

**Use Case**: Quick testing, comparing routed vs unrouted results, using routing with external models.

#### 5.1.2 Pattern B: Coupled Model (Recommended for Calibration)

```python
# Create coupled model
rr_model = SCSUnitHydrograph(area=100, tc=2.5, dt=1.0)
router = NonlinearMuskingumRouter(K=5.0, m=0.8, n_segments=2)
coupled = CoupledModel(rr_model, router, routing_enabled=True)

# Single simulation call
routed_outflow = coupled.simulate(precipitation)

# Access intermediate results
results = coupled.simulate(precipitation, return_components=True)
print(results['direct_runoff'])   # Before routing
print(results['routed_outflow'])  # After routing
```

**Use Case**: Calibration, production workflows, when routing is a standard part of the modeling chain.

#### 5.1.3 Pattern C: Factory Method

```python
# Create model with optional routing via factory
model = create_rainfall_runoff_model(
    model_type='scs_unit_hydrograph',
    area=100,
    tc=2.5,
    dt=1.0,
    routing={
        'method': 'nonlinear_muskingum',
        'K': 5.0,
        'm': 0.8,
        'n_segments': 2
    }

outflow = model.simulate(precipitation)
```

**Use Case**: Configuration-driven workflows, when model setup is read from config files.

### 5.3 Time Step Considerations

The routing module must handle time step consistency between components:

| Requirement | Specification |
|-------------|---------------|
| Time step source | Inherited from RR model (`rr_model.dt`) |
| Unit consistency | K must be in same units as dt |
| Validation | Router validates that dt ≤ K for stability |
| Interpolation | Not supported; input/output have same time step |

```python
class CalibrationEngine:
    """
    Unified calibration engine supporting RR models with optional routing.
    
    This class orchestrates the optimization of model parameters against
    observed data, handling both rainfall-runoff and routing parameters.
    """
    
    def __init__(
        self,
        model: Union['BaseRainfallRunoffModel', 'CoupledModel'],
        objective_function: str = 'nse',
        optimizer: str = 'differential_evolution'
    ):
        """
        Initialize calibration engine.
        
        Parameters
        ----------
        model : model instance
            Either a standalone RR model or CoupledModel
        objective_function : str
            Objective function name: 'nse', 'rmse', 'kge', 'multi'
        optimizer : str
            Optimizer name: 'differential_evolution', 'scipy_minimize',
            'basinhopping', 'dual_annealing'
        """
        pass
    
    def calibrate(
        self,
        precipitation: np.ndarray,
        observed_discharge: np.ndarray,
        parameters_to_calibrate: Optional[List[str]] = None,
        fixed_parameters: Optional[Dict[str, float]] = None,
        **optimizer_kwargs
    ) -> 'CalibrationResult':
        """
        Run calibration.
        
        Parameters
        ----------
        precipitation : np.ndarray
            Input precipitation time series
            
        observed_discharge : np.ndarray
            Observed discharge for comparison
            
        parameters_to_calibrate : list of str, optional
            Which parameters to optimize. If None, all non-fixed parameters.
            Can include both RR and routing parameters.
            Example: ['CN', 'Tc', 'routing_K', 'routing_m']
            
        fixed_parameters : dict, optional
            Parameters to hold constant during calibration.
            Example: {'routing_n_segments': 3}
            
        **optimizer_kwargs
            Additional arguments passed to the optimizer
            
        Returns
        -------
        result : CalibrationResult
            Object containing calibrated parameters, metrics, and diagnostics
        """
        pass
    
    def _build_objective(
        self,
        precipitation: np.ndarray,
        observed: np.ndarray,
        param_names: List[str]
    ) -> Callable:
        """
        Build objective function for optimizer.
        
        Returns
        -------
        objective : callable
            Function f(params_array) -> float (error metric)
        """
        def objective(param_values: np.ndarray) -> float:
            # Convert array to dict
            params = dict(zip(param_names, param_values))
            
            # Handle integer parameters
            if 'routing_n_segments' in params:
                params['routing_n_segments'] = int(round(
                    params['routing_n_segments']
                ))
            
            # Run simulation
            simulated = self.model.simulate_with_parameters(
                precipitation, params
            )
            
            # Calculate objective
            return self._calculate_objective(observed, simulated)
        
        return objective
```

### 6.4 Objective Functions

The calibration framework supports multiple objective functions:

```python
class ObjectiveFunctions:
    """Collection of objective functions for calibration."""
    
    @staticmethod
    def nse(observed: np.ndarray, simulated: np.ndarray) -> float:
        """
        Nash-Sutcliffe Efficiency.
        
        Returns 1 - NSE for minimization (optimal = 0).
        """
        numerator = np.sum((observed - simulated) ** 2)
        denominator = np.sum((observed - np.mean(observed)) ** 2)
        if denominator == 0:
            return 1.0
        return numerator / denominator  # 1 - NSE
    
    @staticmethod
    def rmse(observed: np.ndarray, simulated: np.ndarray) -> float:
        """Root Mean Square Error."""
        return np.sqrt(np.mean((observed - simulated) ** 2))
    
    @staticmethod
    def kge(observed: np.ndarray, simulated: np.ndarray) -> float:
        """
        Kling-Gupta Efficiency.
        
        Returns 1 - KGE for minimization.
        """
        r = np.corrcoef(observed, simulated)[0, 1]
        alpha = np.std(simulated) / np.std(observed)
        beta = np.mean(simulated) / np.mean(observed)
        kge = 1 - np.sqrt((r - 1)**2 + (alpha - 1)**2 + (beta - 1)**2)
        return 1 - kge
    
    @staticmethod
    def pbias(observed: np.ndarray, simulated: np.ndarray) -> float:
        """Percent Bias (absolute value)."""
        return abs(100 * np.sum(simulated - observed) / np.sum(observed))
    
    @staticmethod
    def multi_objective(
        observed: np.ndarray,
        simulated: np.ndarray,
        weights: Dict[str, float] = None
    ) -> float:
        """
        Weighted multi-objective function.
        
        Default weights: NSE=0.5, KGE=0.3, PBIAS=0.2
        """
        if weights is None:
            weights = {'nse': 0.5, 'kge': 0.3, 'pbias': 0.2}
        
        obj = 0.0
        if 'nse' in weights:
            obj += weights['nse'] * ObjectiveFunctions.nse(observed, simulated)
        if 'kge' in weights:
            obj += weights['kge'] * ObjectiveFunctions.kge(observed, simulated)
        if 'pbias' in weights:
            # Normalize PBIAS to 0-1 scale (assume 50% is very bad)
            obj += weights['pbias'] * min(
                ObjectiveFunctions.pbias(observed, simulated) / 50, 1.0
            )
        return obj
```

## 8. Numerical Considerations

### 8.1 Stability Conditions

#### 8.1.1 Time Step Constraint

For numerical stability, the time step should satisfy:

$$\Delta t \leq 2K$$

Where K is the storage constant (or K_segment for subdivided reaches).

Recommendation: $\Delta t \leq K$ for robust stability.

#### 8.1.2 Courant-Like Condition

For subdivided reaches, aim for:

$$\frac{\Delta t}{K_{segment}} \approx 1$$

This can guide the selection of n_segments:

$$n_{segments} \approx \frac{K_{total}}{\Delta t}$$

### 8.2 Handling Edge Cases

#### 8.2.1 Zero or Near-Zero Discharge

The term $Q^{m-1}$ in the derivative becomes problematic when Q approaches zero and m < 1.

**Solution**: Enforce minimum discharge:

```python
Q = max(Q, self.min_discharge)  # Default: 1e-10
```

#### 8.2.2 Negative Discharge

The power function $Q^m$ is undefined for negative Q when m is not an integer.

**Solution**: 
- Validate inputs to ensure non-negative inflows
- Clip computed outflows to non-negative values
- Issue warning if clipping occurs

```python
if Q_new < 0:
    warnings.warn(
        f"Negative discharge computed ({Q_new:.2e}), clipping to zero"
    )
    Q_new = 0.0
```

#### 8.2.3 Non-Convergence

If Newton-Raphson fails to converge:

**Solutions**:
1. Use bisection method as fallback
2. Reduce time step adaptively
3. Use previous time step value as result with warning

```python
def _solve_with_fallback(self, ...):
    try:
        return self._newton_raphson_solve(...)
    except ConvergenceError:
        warnings.warn("Newton-Raphson failed, using bisection fallback")
        return self._bisection_solve(...)
```

### 8.3 Mass Conservation Verification

The routing should conserve mass. Implement verification:

```python
def verify_mass_balance(
    inflow: np.ndarray,
    outflow: np.ndarray,
    dt: float,
    tolerance: float = 0.01
) -> Tuple[bool, float]:
    """
    Verify mass conservation in routing.
    
    Parameters
    ----------
    inflow : np.ndarray
        Inflow time series
    outflow : np.ndarray
        Routed outflow time series
    dt : float
        Time step
    tolerance : float
        Acceptable relative error (default 1%)
        
    Returns
    -------
    is_balanced : bool
        Whether mass is conserved within tolerance
    relative_error : float
        Relative mass balance error
    """
    total_inflow = np.sum(inflow) * dt
    total_outflow = np.sum(outflow) * dt
    
    # Account for storage change
    # Initial storage: S_0 = K * Q_0^m
    # Final storage: S_f = K * Q_f^m
    # Mass balance: Inflow = Outflow + (S_f - S_0)
    
    if total_inflow == 0:
        return total_outflow == 0, 0.0
    
    relative_error = abs(total_inflow - total_outflow) / total_inflow
    
    return relative_error < tolerance, relative_error
```

## Appendix A: Mathematical Derivations

### A.1 Derivation of Newton-Raphson Function

Starting from the discretized mass balance:

$$S^{n+1} - S^n = \Delta t \left( \frac{I^n + I^{n+1}}{2} - \frac{Q^n + Q^{n+1}}{2} \right)$$

Substituting $S = KQ^m$:

$$K(Q^{n+1})^m - K(Q^n)^m = \Delta t \left( \bar{I} - \frac{Q^n + Q^{n+1}}{2} \right)$$

Rearranging to form $f(Q^{n+1}) = 0$:

$$K(Q^{n+1})^m + \frac{\Delta t}{2}Q^{n+1} = K(Q^n)^m - \frac{\Delta t}{2}Q^n + \Delta t \bar{I}$$

$$f(Q) = K Q^m + \frac{\Delta t}{2}Q - \left[ K(Q^n)^m - \frac{\Delta t}{2}Q^n + \Delta t \bar{I} \right] = 0$$

### A.2 Derivative for Newton-Raphson

$$f'(Q) = \frac{df}{dQ} = Km Q^{m-1} + \frac{\Delta t}{2}$$
